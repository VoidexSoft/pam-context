**Modify `_smart_search()` in `src/pam/agent/agent.py`:**

1. **Add two new search coroutines** inside `_smart_search()`, after the existing `_es_search_coro` and `_graph_search_coro` definitions:

```python
async def _entity_vdb_search_coro() -> list[dict]:
    if self.vdb_store is None:
        return []
    return await self.vdb_store.search_entities(
        query_embedding=es_query_embedding,
        top_k=settings.smart_search_entity_limit,
    )

async def _rel_vdb_search_coro() -> list[dict]:
    if self.vdb_store is None:
        return []
    return await self.vdb_store.search_relationships(
        query_embedding=graph_query_embedding,
        top_k=settings.smart_search_relationship_limit,
    )
```

**Key optimization (per research open question 4):** Reuse query embeddings between searches that use the same query text. The ES segment search already embeds `es_query`. Reuse that embedding for entity VDB search (both use low-level keywords). Create a single embedding for `graph_query` and reuse it for relationship VDB search (both use high-level keywords). This halves the embedding API calls per search.

Refactor the embedding to happen before the gather:
```python
# Embed both queries upfront (reuse for VDB searches)
es_query_embedding, graph_query_embedding = (await self.embedder.embed_texts([es_query, graph_query]))
# or if the texts are the same, embed once
query_embeddings = await self.embedder.embed_texts([es_query, graph_query])
es_query_embedding = query_embeddings[0]
graph_query_embedding = query_embeddings[1]
```

Then pass `es_query_embedding` to the existing `_es_search_coro` (modify it to accept the pre-computed embedding instead of embedding inside the coroutine).

2. **Extend `asyncio.gather`** to include all 4 coroutines:

es_result, graph_result, entity_vdb_result, rel_vdb_result = await asyncio.gather(
    _es_search_coro(),
    _graph_search_coro(),
    _entity_vdb_search_coro(),
    _rel_vdb_search_coro(),
    return_exceptions=True,
)
