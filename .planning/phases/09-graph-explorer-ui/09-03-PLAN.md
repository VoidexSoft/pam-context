**1. Create `web/src/components/graph/IngestionDiff.tsx`:**

This component displays ingestion diff data and applies color coding to canvas nodes.

Props:
- `onApplyDiff: (diffColors: Map<string, string> | null) => void` -- Callback to apply or clear color coding on canvas nodes. The map keys are entity names, values are hex colors.
- `className?: string`

State:
- `syncLogs: SyncLogEntry[]` -- Fetched from `getGraphSyncLogs({ limit: 10 })`
- `selectedLogId: string | null` -- Currently selected log for diff view
- `filterMode: "filtered" | "highlighted"` -- Toggle between filtered subgraph and full graph with highlights (default: "filtered" per user decision)
- `loading: boolean`

On mount: Fetch `getGraphSyncLogs({ limit: 10 })` and store in state.

UI layout:
- Collapsible section header "Ingestion Changes" with chevron icon
- When expanded:
  - Dropdown/select for choosing a sync log entry (show document name + timestamp)
  - Toggle buttons: "Filtered" | "Highlighted" (default: Filtered)
  - When a log is selected, show changes grouped by type:
    - "Added" section (green heading): list entity names from `details.added`
    - "Changed" section (yellow heading): list entity names from `details.modified` with change details
    - "Invalidated" section (red heading): list entity names from `details.removed_from_document`

Diff color constants (from RESEARCH.md):
```typescript
const DIFF_COLORS = {
  added: "#22c55e",
  changed: "#eab308",
  invalidated: "#ef4444",
};
```

When a log is selected, build a `Map<string, string>` of entity name -> color:
- `details.added` entries -> DIFF_COLORS.added (green)
- `details.modified` entries -> DIFF_COLORS.changed (yellow)
- `details.removed_from_document` entries -> DIFF_COLORS.invalidated (red)

Call `onApplyDiff(colorMap)` when a log is selected. Call `onApplyDiff(null)` when deselected or cleared.

When no sync logs exist: show "No ingestion history available."

Read the existing `useMarkdownComponents.tsx` file first to understand the current structure.

The agent returns entity references as markdown links like `[Entity Name](/graph/explore?entity=EntityName)` in response text. The existing custom anchor renderer needs to:

1. Check if the href starts with `/graph/explore` (internal graph link)
2. If it's an internal graph link AND `VITE_GRAPH_ENABLED` is true:
   - Use `useNavigate()` from `react-router-dom` to navigate (SPA navigation, no full reload)
   - Add visual styling to distinguish entity links: underline + a subtle graph icon or distinct color (e.g., indigo to match the Graph nav theme)
3. If it's an external link or graph is disabled: use normal `<a>` tag behavior (existing logic)

Implementation approach:
- Import `useNavigate` from `react-router-dom` (if not already imported)
- In the anchor component (or `renderAnchor` function), add a check:
  ```tsx
  if (href?.startsWith("/graph/explore")) {
    const graphEnabled = import.meta.env.VITE_GRAPH_ENABLED === "true";
    if (graphEnabled) {
      return (
        <button
          onClick={(e) => {
            e.preventDefault();
            navigate(href);
          }}
          className="text-indigo-600 hover:text-indigo-800 underline cursor-pointer inline"
          title="View in Graph Explorer"
        >
          {children}
        </button>
      );
    }
  }
  ```
- Use a `<button>` styled as a link (or a regular `<a>` with onClick that calls `navigate`) to avoid accessibility issues with fake anchors.

Note: The agent's graph tools already return entity names in responses. In the future, the agent prompt could be updated to include graph explorer links, but for now this handles the link rendering when they appear.
