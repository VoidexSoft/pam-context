**3. Add `_smart_search()` method to RetrievalAgent class:**

```python
async def _smart_search(self, input_: dict) -> tuple[str, list[Citation]]:
```

Implementation steps:

**Step A: Extract keywords**
- Call `extract_query_keywords(self.client, query)` where query = `input_["query"]`
- If extraction raises any exception, return an error message to the agent: `"Keyword extraction failed: {error}. Try search_knowledge or search_knowledge_graph instead."` with empty citations. Per user decision: return error, let agent retry or fall back.

**Step B: Prepare search queries**
- ES query: join low_level_keywords with space. If empty, fall back to the original query (per research recommendation -- avoids empty result pitfall).
- Graph query: join high_level_keywords with space. If empty, fall back to the original query.
- Read limits from config: `settings = get_settings(); es_limit = settings.smart_search_es_limit; graph_limit = settings.smart_search_graph_limit`

**Step C: Define async search coroutines**
- `_es_search_coro`: embed the ES query via `self.embedder.embed_texts([es_query])`, then call `self.search.search(query=es_query, query_embedding=embeddings[0], top_k=es_limit, source_type=self._default_source_type)`. Return the results list.
- `_graph_search_coro`: call `search_graph_relationships(graph_service=self.graph_service, query=graph_query)` from `pam.graph.query`. Return the raw text result.
- If `self.graph_service is None`, skip graph search: set graph coroutine to return empty result immediately (or use a simple async function that returns an empty string).

**Step D: Run concurrently with asyncio.gather**
- `es_result, graph_result = await asyncio.gather(es_coro, graph_coro, return_exceptions=True)`
- Check each result with `isinstance(result, Exception)`:
  - If ES failed: set `es_results = []`, add warning `"es_backend_failed"`
  - If graph failed: set `graph_text = ""`, add warning `"graph_backend_failed"`

**Step E: Format document_results from ES**
- For each ES search result, build a dict-like entry: source label, content, content_hash (SHA-256 of content for dedup)
- Build `Citation` objects from ES results (same pattern as `_search_knowledge`)
- Collect content hashes in a set for dedup

**Step F: Format graph_results**
- The graph search returns a formatted text string (from `search_graph_relationships`). Since graph results are already text-formatted with relationship structure (source entity, target entity, relationship type), include the text as-is.
- For dedup: the graph result text is a single formatted block, not individual items, so content hash dedup applies at the chunk level. In practice, dedup across ES and graph is rare since they return different formats. Simply include the graph text.

**Step G: Backfill logic**
- If ES returned fewer than `es_limit`, graph gets extra quota: `graph_effective_limit = graph_limit + (es_limit - len(es_results))`
- If graph returned fewer results than `graph_limit`, ES gets extra quota (but ES results are already fetched, so this backfill only applies to the initial search -- for simplicity, just note the imbalance in the total count).
- For Phase 12, backfill is best-effort: if one source underperforms, the other source's full results compensate. No need to re-query.

**Step H: Build final output string**
- Structure the result as a formatted text block that includes:
  ```
  Keywords extracted:
  - High-level: {", ".join(keywords.high_level_keywords)}
  - Low-level: {", ".join(keywords.low_level_keywords)}

  ## Document Results ({n} results)
  {formatted ES results with [Result N] Source: ... prefix, same format as _search_knowledge}

  ## Graph Results
  {graph text from search_graph_relationships, or "No graph results found." if empty}

  {optional: "Warning: {backend} search was unavailable, showing partial results." if one backend failed}
  ```
- Return the formatted string and the collected Citation list

**4. Update _execute_tool dispatch:**
Add `smart_search` case before the existing tool checks:

- Dedup by content hash favors ES version (ES carries citations)
- If keyword extraction fails, return error (not fallback to default query)
- If one backend fails, return partial results with warning field
- All three search tools described equally in system prompt -- no forced preference
