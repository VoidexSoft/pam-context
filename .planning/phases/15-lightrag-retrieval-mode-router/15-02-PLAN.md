<objective>
Wire the query classifier into smart_search so that the classified mode controls which retrieval paths actually execute, add the optional mode parameter to the SMART_SEARCH_TOOL definition, extend AgentResponse and ChatResponse with mode metadata, and add observability logging. This completes the LightRAG-inspired retrieval mode router.

Purpose: Reduces average retrieval latency by skipping irrelevant search paths for 40%+ of queries (factual mode is ~3-4x faster, entity mode ~2x faster), while preserving full coverage for ambiguous queries via hybrid fallback. Mode decisions are visible to API consumers and logged for tuning.

Output: Modified agent.py, tools.py, chat.py, and integration tests.
</objective>

The current _smart_search method processes input_, extracts keywords, runs 4 concurrent searches, handles errors, and assembles context. The modifications are:

**4a. After Step A (keyword extraction), before Step B (query prep), add classification:**

# Step A2: Classify query mode
forced_mode_str = input_.get("mode")
if forced_mode_str:
    try:
        mode = RetrievalMode(forced_mode_str)
        classification = ClassificationResult(mode=mode, confidence=1.0, method="forced")

**4b. Replace the current Step D (run all 4 searches) with mode-conditioned execution:**

Define noop coroutines as local async functions:

```python
async def _noop_list() -> list:
    return []

async def _noop_str() -> str:
    return ""
```

Then build the coroutine list based on mode. The mapping is:
- **FACTUAL:** ES only — skip graph, entity VDB, relationship VDB
- **ENTITY:** ES + entity VDB — skip graph, relationship VDB
- **CONCEPTUAL:** ES + graph + relationship VDB — skip entity VDB (include ES because conceptual queries still benefit from document context, per open question #2 in research)
- **TEMPORAL:** All 4 paths — temporal queries need broad context
- **HYBRID:** All 4 paths — default, no skipping

```python
# Step D: Run searches concurrently (mode-conditioned)
if mode == RetrievalMode.FACTUAL:
    es_coro = _es_search_coro()
    graph_coro = _noop_str()
    entity_vdb_coro = _noop_list()
    rel_vdb_coro = _noop_list()
elif mode == RetrievalMode.ENTITY:
    es_coro = _es_search_coro()
    graph_coro = _noop_str()
    entity_vdb_coro = _entity_vdb_search_coro()
    rel_vdb_coro = _noop_list()
elif mode == RetrievalMode.CONCEPTUAL:
    es_coro = _es_search_coro()
    graph_coro = _graph_search_coro()
    entity_vdb_coro = _noop_list()
    rel_vdb_coro = _rel_vdb_search_coro()
else:  # TEMPORAL or HYBRID — all paths
    es_coro = _es_search_coro()
    graph_coro = _graph_search_coro()
    entity_vdb_coro = _entity_vdb_search_coro()
    rel_vdb_coro = _rel_vdb_search_coro()

es_result, graph_result, entity_vdb_result, rel_vdb_result = await asyncio.gather(
    es_coro, graph_coro, entity_vdb_coro, rel_vdb_coro,
    return_exceptions=True,
)
```
