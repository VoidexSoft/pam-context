### Integration Issues

**MODE-03 Frontend Gap (Medium — not a requirement violation)**
- Backend: `AgentResponse.retrieval_mode`, `ChatResponse.retrieval_mode`, and SSE `done` event all emit mode metadata correctly
- Frontend: `StreamEvent.metadata` TypeScript type (`client.ts:146-151`) and `ChatResponse` interface (`client.ts:17-28`) do not declare `retrieval_mode`/`mode_confidence`
- `useChat.ts` `done` handler reads only `token_usage` and `latency_ms`
- **Impact:** Mode data flows to the browser wire format but is silently dropped by the TypeScript layer. No UI renders mode information.
- **Assessment:** MODE-03 requirement ("Mode classification is logged in agent response metadata for observability and tuning") is satisfied at the backend level — structlog logging and API response metadata are complete. The frontend gap is a quality improvement opportunity, not a requirement violation.
